API

API Description


Approved users of the Envitus IoT Suite API can access the following details:

  • /device/sensor/stats
  • /device/sensor/stats/count
  • /device/sensor/livedata/v1
  • /device/sensor/livedata/v1/count

Request API Access


Envitus grants only a limited number of API keys, and allow access to the Content API for purposes of:

  • Third Party integrations for immediate client only

Envitus grants only a limited number of API keys, and does not allow access to the Content API for purposes of:

  • Academic research
  • Any use not associated with a client application

API key will allow limited access of 500 calls per-day/ per device to the API - this key is to be used for development purposes only.

The API key is provided to the concerned authority on premise instalation installation.


Documentation


No API URL Comments
1 /device/sensor/stats Use this API to return the daily/monthly/yearly statistics data for a device.
Statistics data includes will include min, max, number of packets received for each device parameter.
2 /device/sensor/stats/count This API will return the raw data for the requested devices.
Data Ordering will be based on received time that is recent data will be put first.
This API will also support start and end time value while fetching data.
3 /device/sensor/livedata/v1 This API will return the raw data for the requested devices.
Data Ordering will be based on received time that is recent data will be put first.
This API will also support start and end time value while fetching data.
4 /device/sensor/livedata/v1/count Use this API to get the total number of raw data captured for given devices.

API Details


1. Api: /device/sensor/stats


Parameter supported


Parameter Name Optional/Must Purpose
userId Must For now use value ‘admin’
authPassword Must For now use value ‘admin’
deviceIds Must This parameter can contain the comma separated device identifiers or a single device id
limit Must This parameter can contain the comma separated device identifiers or a single device id
timeStart,timeEnd optional This indicates the time epoch value of the start and end time for collecting device statistics over a period.
If not given, api will return data in latest to oldest order.
offset optional By default it will be 0
timeFrame Must timeFrame can take either following values daily monthly, yearly.
params optional This can be a comma separated list of parameters.
For example if only temperature and pressure is needed then give ‘temperature, pressure’.

By default API will consider all parameters

Example

http://117.247.186.190:4001/device/sensor/stats?userId=admin&authPassword=admin&deviceIds=70b3d5fffe098009&timeFrame=daily&limit=2


For asking data for device with ID 70b3d5fffe098009 and time frame with ‘daily’.
API will return daily device statistics sorted in latest to oldest order.
limit parameter tells API to return only 2 records as part of ‘dailtStat’ list.
But there is no limit on this value.

API Result

{"statPerDeviceId":[{"deviceId":"70b3d5fffe098009","stat":{"dailyStat":
[{"paramName":"temperature","key":"22.11.2018","statParams":
{"sum":10.139999999999999,"count":6,"min":1.69,"max":1.69}},
{"paramName":"rain","key":"22.11.2018","statParams":{"sum":120,"count":6,"min":20,"max":20}}]}}]}}

2. Api: /device/sensor/stats/count


Parameter supported


Parameter Name Optional/Must Purpose
userId Must For now use value ‘admin’
authPassword Must For now use value ‘admin’
deviceIds Must This parameter can contain the comma separated device identifiers or a single device id
timeStart,timeEnd optional This indicates the time epoch value of the start and end time for collecting device statistics over a period.
If not given, api will return data in latest to oldest order.
timeFrame Must timeFrame can take either following values daily monthly, yearly.
params optional This can be a comma separated list of parameters.
For example if only temperature and pressure is needed then give ‘temperature, pressure’.

By default API will consider all parameters

Example

http://117.247.186.190:4001/device/sensor/stats/count?userId=admin&authPassword=admin&
deviceIds=70b3d5fffe098009&timeFrame=daily,monthly,yearly


This request will return the number of records stored for statistics data for device '70b3d5fffe098009'.
‘statistics record count’ for device '70b3d5fffe098009' .
Record count will be calculated based on the ‘params’ and ‘timeStart’ and ‘timeEnd’.
If ‘timeStart’ and ‘timeEnd’ parameter are absent, system will calculate record count for complete data set.

API Result

{"status":"ok","errorCode":0,"message":null,"data":{"statCountPerDeviceId":
[{"deviceId":"70b3d5fffe098009","stat":{"dailyStat":23,"monthlyStat":23,"yearlyStat":23}}]}}
